# Настройка системы заказов 3Dxob

Заказы с сайта приходят в Google-таблицу и (опционально) в Telegram. Бэкенда нет — всё работает на бесплатном Google Apps Script.

## Шаги настройки

### 1. Создать Google-таблицу

1. Зайдите на https://sheets.google.com
2. Создайте новую пустую таблицу
3. Назовите её, например, «Заказы 3Dxob»

### 2. Подключить Apps Script

1. В таблице: **Расширения → Apps Script** (Extensions → Apps Script)
2. Откроется редактор кода. Удалите содержимое файла `Code.gs`
3. Скопируйте всё содержимое файла `Code.gs` из этого репозитория и вставьте в редактор
4. Нажмите **💾 Сохранить** (Ctrl+S)

### 3. Задать стартовый номер заказа

1. В Apps Script слева: **⚙️ Project Settings**
2. Прокрутите вниз до **Script Properties**
3. Нажмите **Add script property**:
   - Property: `lastOrderNumber`
   - Value: `986` (тогда первый заказ будет `3DX-0987`; можно поставить любое число)
4. **Save script properties**

### 4. Развернуть как Web App

1. В Apps Script справа сверху: **Deploy → New deployment**
2. Шестерёнка **Select type** → **Web app**
3. Заполните:
   - **Description**: `3Dxob orders v1`
   - **Execute as**: `Me` (ваш аккаунт)
   - **Who has access**: `Anyone` (это нужно, чтобы сайт мог отправлять)
4. **Deploy**
5. Google попросит разрешения — нажмите **Authorize access**, выберите аккаунт, на «This app isn't verified» → **Advanced → Go to (project name)** → **Allow**
6. Скопируйте **Web app URL** — он выглядит как `https://script.google.com/macros/s/AKfyc.../exec`

### 5. Прописать URL в сайт

Откройте файл `js/app.js` и замените:

```js
window.ORDER_WEBHOOK_URL = 'https://script.google.com/macros/s/PASTE_DEPLOY_ID_HERE/exec';
```

на скопированный из шага 4 URL.

Закоммитьте и запушьте — GitHub Pages обновится сам.

### 6. Проверить

1. Откройте сайт, положите что-то в корзину, нажмите «Оформить заказ»
2. Вы должны попасть на страницу `success.html` с номером заказа
3. В Google-таблице должен появиться лист «Заказы» с новой строкой

## Telegram-уведомления (опционально)

### Создание бота

1. В Telegram найдите **@BotFather**
2. Команда `/newbot` → задайте имя и username → получите **токен** (`123456:AAH...`)
3. Найдите **@userinfobot**, нажмите `/start` → получите свой **chat_id** (число)
4. **ВАЖНО**: найдите своего нового бота в поиске и напишите ему `/start`. Без этого бот не сможет отправлять сообщения.

### Прописываем в Code.gs

1. В Apps Script найдите строки:
   ```js
   var TELEGRAM_BOT_TOKEN = '';
   var TELEGRAM_CHAT_ID   = '';
   ```
2. Подставьте свои значения:
   ```js
   var TELEGRAM_BOT_TOKEN = '123456:AAH...';
   var TELEGRAM_CHAT_ID   = '987654321';
   ```
3. Сохраните (Ctrl+S) — переразвёртывание не нужно, изменения подхватятся сразу

### Тест уведомления

1. В Apps Script сверху выберите функцию `testTelegram`
2. Нажмите **Run**
3. Должно прийти сообщение «🛒 НОВЫЙ ЗАКАЗ 3DX-TEST...»

## Работа с заказами в таблице

Колонки:

| Поле | Что значит |
|---|---|
| Номер заказа | `3DX-0987`, `3DX-0988`, ... |
| Дата | Время оформления |
| Статус | `⏳ Ждём сообщения` — клиент ещё не написал; `✅ Подтверждён` — менять вручную после контакта в чате |
| ФИ / Контакт / Email | `—` (анонимный заказ — клиент назовётся в мессенджере) |
| Доставка | `Уточняется в чате` |
| Товары | Состав заказа списком |
| Кол-во | Общее количество позиций |
| Сумма | Итог в рублях |
| User-Agent | Браузер клиента (для отладки) |
| Referer | Откуда пришёл клиент |

После того, как клиент напишет в Telegram/MAX и подтвердит заказ — поменяйте «Статус» вручную, добавьте ФИ/контакты/адрес доставки в соответствующие колонки.

## Защита от спама (уже встроена)

- **Honeypot**: скрытое поле в форме, если бот заполнит — заказ молча игнорируется
- **Rate limit**: не больше 10 заказов с одного браузера за час
- **Дневной лимит**: не больше 200 заказов в сутки от всех
- **Дедупликация**: одинаковый заказ за 60 секунд схлопывается в один

## Что делать, если сервер недоступен

Сайт автоматически переключится в fallback-режим:
- Заказу присвоится локальный номер (тоже `3DX-XXXX`)
- На success-странице покажется жёлтое предупреждение
- Клиент получит чёткое указание написать в Telegram/MAX, чтобы вы оформили заказ вручную

## Обновление кода Apps Script

Если правите `Code.gs`:

1. Сохраните изменения (Ctrl+S)
2. **Deploy → Manage deployments**
3. У активного деплоя нажмите **✏️ карандаш**
4. **Version → New version → Deploy**
5. URL остаётся прежним

## Что менять под другие проекты

| Где | Что |
|---|---|
| `js/app.js` | `ORDER_WEBHOOK_URL`, `TG_URL`, `MAX_URL`, `ORDER_PREFIX`, `ORDER_START` |
| `Code.gs` | `ORDER_PREFIX`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `SHEET_NAME` |
| `pages/success.html` | константа `SITE` (домен в готовом сообщении) |
| Apps Script Properties | `lastOrderNumber` (стартовое значение) |
